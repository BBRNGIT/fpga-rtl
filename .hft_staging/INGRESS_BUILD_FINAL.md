## Build Methodology Enforced

### Emitter-First Workflow

Every component follows the three-step pipeline:

1. **Emitter** (`gen_<module>_net.py`) — Python script describing the circuit
2. **Netlist** (`<module>.net.json`) — Declarative circuit spec (committed, validates integrity via checksum)
3. **Generated Device Code** (`<module>_gen.h`) — Branchless C, generated by gennet, **COMMITTED and GRADUATED** (immutable vault copy)

Example (fifo_rx, 858 KB circuit):
```
gen_fifo_rx_net.py  → fifo_rx.net.json → gennet.py → fifo_rx_gen.h
(emitter)           (spec)              (generator)  (immutable circuit)
```

### Pre-Commit Enforcement

- **thin-test law:** Tests are power-on + display only. No orchestration, clock stepping, data injection, or wire reads. (Hook: `.githooks/check_thin_tests.sh`)
- **build-sequence law:** Hand-written source files contain ZERO device logic (`cell_*()`, `*_tick` bodies). Only emitter + generated code. (Hook: `.githooks/check_generated.sh`, updated to enforce committed circuit byte-match)
- **drift guard:** No floats, no heap, no module-barrier violations, no invented vocabulary. (Hook: `.githooks/check_drift.sh`)

### Gate Stages (All Pass)

**Stage 1: Validate** — Netlist schema, single-writer check, no-overlap, no-floating, module-barrier compliance  
**Stage 2: Build** — Emitter → netlist → gennet → C → compile  
**Stage 3: 2b (gate-level arithmetic)** — No native `+`/`-`/`*` in generated tick; only `cell_addsub`, `cell_eqmask`, `cell_mux` (structural cells)  
**Stage 4: 2c (no hand-written device)** — Grep confirms no `cell_*()` or `*_tick` bodies in source files (only in `*_gen.h`)  
**Stage 5: Clean-room build** — Rebuild from committed HEAD 3× independently; byte-identical binaries  

All **9 components pass all 5 gates**.

## Architectural Compliance

### Module Barrier Law
✅ **Enforced.** Each component samples its predecessor's **published output window** (wire lanes, seam lanes, head slot), NOT its internal registers.

Example: NIC reads `SEAM_*` lanes from adapter via wire bus; does NOT raw-read adapter's internal price history.

### Clock Domain Separation
✅ **Absolute.** Four independent domains:
- **MAC clock** (125 MHz): NIC ingress, FIFO write side
- **Internal clock** (250 MHz): Pipeline, FIFO read side
- **TAI clock** (timestamp source): Separate oscillator
- **CDC seam:** FIFO_RX's gray pointers + dual 2-FF sync

No raw cross-domain reads. All multi-bit counters cross via CDC.

### Two-Clock Model (Validated)
✅ **MAC = sample rate** (when wire is read, packet is stamped)  
✅ **TAI = timestamp value** (separate clock, read via tai_cdc's CDC output)  
✅ **No discipline** (taiosc IS the reference, tai plain counter, no PPS/PI)  

### Branchless Data Path
✅ **All stages.**
- Adapter: fixed-point arithmetic
- Wire: addressed lanes (no compute)
- NIC: dedup via bitwise ops (`cell_eqmask` + `cell_mux`)
- FIFO: gray encode/decode via XOR/shift, pointer step via `cell_addsub`, flags via `cell_eqmask`

No loops over bits, no nested conditionals, no function calls in tick.

### Gate-Level Arithmetic (RTL Fidelity)
✅ **Enforced.** Device datapath is **structural cells only**:
- `cell_addsub` — carry-chain ripple (not native `+`)
- `cell_eqmask` — comparison logic (not native `==`)
- `cell_mux` — select gates (not native `?:`)
- No native `+`/`-`/`*` in generated tick

Example: FIFO pointer advance via `cell_addsub` ripple-carry (slower sim, faithful RTL).
