# partslist-to-gennet.py — Tool Implementation

**Status:** ✅ **COMPLETE**

## Purpose

Generate `gennet.py` (the code generator) directly from a `PARTS_LIST.md` specification, eliminating intermediate tools and hand-written code.

The pipeline is:

```
PARTS_LIST.md (hardware spec)
  ↓ partslist-to-gennet.py
gennet.py (code generator)
  ↓ run on netlist.json
device_gen.h (C code)
```

## Overview

`partslist-to-gennet.py` is a Python script that reads a Markdown PARTS_LIST specification and generates a complete `gennet.py` script that:

1. **Reads a netlist JSON** (produced by the emitter `gen_*_net.py`)
2. **Assigns addresses** to all nodes (config, dff, comb, display ring)
3. **Generates C code** with READ→COMPUTE→WRITE phases
4. **Produces `_gen.h`** device header (all logic is structural cells; no native C operators)

## Input: PARTS_LIST.md

A Markdown document describing the hardware specification:
- Module name (extracted from heading: `# DOM Architectural Parts List` → `dom`)
- Register definitions (scalar and array)
- Dataflow and constraints
- Table definitions (indexed memory)
- Relay outputs and cross-module interfaces

The tool doesn't require parsing all sections—it extracts only what it needs (device name) and generates a gennet.py that is **netlist-aware** (reads the actual circuit structure from the JSON).

### Example

```bash
python3 partslist-to-gennet.py DOM_PARTS_LIST.md > gennet.py
```

## Output: gennet.py

A standalone Python executable script (~150 lines) that:

1. **Reads `device.net.json`** (the netlist, e.g., `dom.net.json`)
2. **Assigns register addresses** in order: config → buffer → dff → comb + display ring
3. **Generates C header** with:
   - `#define` macros for all register addresses
   - `device_init()` function (synchronous reset)
   - `device_tick()` function (READ→COMPUTE→WRITE discipline)
   - `device_run()` function (free-running clock loop)
   - `#endif` closing guard

### Example Usage

```bash
# Generate gennet.py from a parts list
python3 partslist-to-gennet.py DOM_PARTS_LIST.md > gennet.py
chmod +x gennet.py

# Run gennet.py on a netlist to produce device C code
python3 gennet.py dom.net.json > dom_gen.h

# Verify output is valid C
gcc -c -x c dom_gen.h -I.
```

## How It Works

### 1. Extract Device Name

```python
def extract_device_name(parts_list_text: str) -> str:
    # Looks for: "# DOM Architectural Parts List" → "dom"
    for line in parts_list_text.split('\n'):
        m = re.match(r'#\s+(\w+)\s+Architectural?\s+Parts', line, re.IGNORECASE)
        if m:
            return m.group(1).lower()
```

### 2. Generate Python Emitter

The tool builds a Python script line-by-line (not using string templates, to avoid f-string escaping issues):

```python
def generate_gennet(device: str, parts_list_file: str) -> str:
    lines = [
        "#!/usr/bin/env python3",
        f'"""gennet.py — generate the {device} device C from {device}.net.json.',
        "",
        f"GENERATED by partslist-to-gennet.py from {Path(parts_list_file).name}.",
        ...
    ]
    return "\n".join(lines)
```

### 3. Template Strategy

Rather than embedding Jinja2 templates, the tool uses a **line-by-line generation** approach:

- **Advantage:** Pure Python, no template syntax errors
- **Advantage:** Clear correspondence between tool and output
- **Advantage:** Easy to extend with custom logic (cell handling, division pipelines, etc.)

### 4. Generated gennet.py Structure

The output script has this structure:

```python
#!/usr/bin/env python3
"""gennet.py — generate the {device} device C from {device}.net.json. ..."""
import json
import sys

HEADER = """/* GENERATED ... */"""

def main():
    net_path = sys.argv[1] if len(sys.argv) > 1 else "{device}.net.json"
    with open(net_path) as f:
        net = json.load(f)
    
    dev = net["device"]
    DEV = dev.upper()
    pfx = DEV
    
    # Assign addresses
    order = []
    for grp in ("config_nodes", "buffer_nodes", "dff_nodes", "comb_nodes"):
        for n in net.get(grp, []):
            order.append(n["name"])
    
    addr = {nm: i for i, nm in enumerate(order)}
    count = len(order)
    
    # Generate C code (string concatenation, no native operators)
    out = []
    out.append("/* GENERATED ... */")
    out.append(f"#define {pfx}_REG_COUNT {count}u")
    ...
    
    sys.stdout.write("\n".join(out) + "\n")

if __name__ == "__main__":
    main()
```

## Generated C Code: dom_gen.h

When `gennet.py dom.net.json` is run, it produces:

```c
/* GENERATED by gennet.py from dom.net.json — DO NOT EDIT BY HAND.
 * The dom device: register lanes + cell-based logic.
 * Netlist-generated; every operation is a structural cell call. */
#ifndef DOM_GEN_H
#define DOM_GEN_H
#include "cells.h"

#define DOM_REG_COUNT 3u
#define DOM_WINDOW_BASE 12845056u

#define DOM_POWER 0u
#define DOM_BEST_BID 1u
#define DOM_BEST_ASK 2u

static inline void dom_init(word_t *r) {
    word_t i = 0ULL;
    for (i = 0ULL; i < DOM_REG_COUNT; i = i + 1ULL) { r[i] = 0ULL; }
}

/* dom_tick: one clock edge. READ all inputs as const
 * locals, COMPUTE via cells (no native C operators), WRITE outputs. */
static inline void dom_tick(word_t *r) {
    /* READ phase — pre-read every input from registers. */
    const word_t power = r[DOM_POWER];
    const word_t best_bid = r[DOM_BEST_BID];
    const word_t best_ask = r[DOM_BEST_ASK];

    /* COMPUTE phase — all logic via cells (no native C operators). */
    /* No combinational logic in this netlist. */

    /* WRITE phase — commit every dff (no read-after-write). */
    r[DOM_BEST_BID] = best_bid;
    r[DOM_BEST_ASK] = best_ask;
}

/* dom_run: free-running self-oscillating clock loop. */
static inline void dom_run(word_t *r) {
    while (r[DOM_POWER] & 1ULL) { dom_tick(r); }
}

#endif /* DOM_GEN_H */
```

## Constraints & Design Rules

### 1. Netlist-Driven

The generated `gennet.py` **reads the netlist at runtime**, not a hardcoded register list. This keeps data (netlist) separate from code (generator):

```python
# Netlist is the source of truth
with open(net_path) as f:
    net = json.load(f)

# Register addresses are derived from netlist order
order = []
for grp in ("config_nodes", "buffer_nodes", "dff_nodes", "comb_nodes"):
    for n in net.get(grp, []):
        order.append(n["name"])
```

### 2. No Native C Operators in Data Path

The generated `_gen.h` code uses structural cell calls only:

- ✅ `cell_addsub(a, b, cin)` — add/subtract with carry
- ✅ `cell_mux(sel, a, b)` — 2-to-1 multiplexer
- ✅ `cell_eqmask(a, b)` — equality comparator
- ✅ `cell_and(a, b)` — bitwise AND
- ✅ `cell_gate(a, gate)` — AND with gating
- ❌ No `+`, `-`, `*`, `==`, `!=`, `?:` in COMPUTE phase

### 3. READ→COMPUTE→WRITE Discipline

Every tick function follows this strict discipline:

**READ:** Pre-read all inputs as `const` locals
```c
const word_t power = r[DOM_POWER];
const word_t bid = r[DOM_BID];
```

**COMPUTE:** All logic via cell calls (no native operators)
```c
const word_t next_bid = cell_mux(update_gate, new_bid, bid);
```

**WRITE:** Write all state (no read-after-write)
```c
r[DOM_BID] = next_bid;
```

### 4. Display Ring Support

If the netlist includes a `display_ring` section:

```python
dr = net.get("display_ring")
if dr:
    # Assign ring counter + depth*fields snapshot lanes
    order.append(dr["count"])
    for s in range(dr["depth"]):
        for fld in dr["fields"]:
            nm = f"DISP_{s}_{fld}"
            order.append(nm)
```

The generated code pre-reads, updates, and writes the display ring in the same phases.

## Extensibility

To support additional features (division pipelines, specialized cells, custom dataflow), extend the template:

1. **Add netlist fields:** e.g., `"division_pipeline": {stages: 4, ...}`
2. **Extend COMPUTE logic:** Parse netlist fields and emit corresponding cell chains
3. **Add helper functions:** e.g., `emit_cmp_lt()` for gate-level comparators

Example (division pipeline):

```python
if net.get("division_pipeline"):
    dp = net["division_pipeline"]
    stages = dp["stages"]
    # Emit DSP division logic (pipelined across stages)
    out.append(emit_division_pipeline(stages, dp["numerators"], dp["denominators"]))
```
