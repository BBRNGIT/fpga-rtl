# gennet_device.py — Universal Device C Generator

**gennet_device.py** is a universal code generator that produces complete, compilable C device headers from FPGA netlist JSON specifications. It is the core tool for the C-as-RTL build pipeline.

## Purpose

Given a valid `device.net.json` netlist, gennet_device.py synthesizes `device_gen.h` — a complete, self-contained device model with:

- Device constants (register counts, window base address, full address map)
- Initialization function (synchronous reset to all-zeros)
- Per-tick function with strict READ→COMPUTE→WRITE phase separation
- Gate-netlist comparators (cmp_le, cmp_lt, cmp_ge, cmp_gt) as needed
- All combinational cell instantiations (AND, OR, MUX, ADDSUB, etc.)
- All sequential DFF writes (no read-after-write hazards)
- Optional free-running clock loop (power-bit gated)
- Optional display ring snapshot buffer

## Usage

```bash
python3 gennet_device.py <netlist.net.json> > <device>_gen.h
```

### Example

```bash
cd .hft_staging/adapter
python3 ../../tools/generators/gennet_device.py adapter.net.json > adapter_gen.h
gcc -c adapter_gen.h  # Verify it compiles
```

## Design Principles

### 1. Netlist is the Source of Truth

The netlist JSON fully specifies the device structure:
- All registers (config, buffer, DFF, combinational, display ring)
- All operations (gates, arithmetic, comparators)
- Special sections (clock, pos, writeout, display_ring)

The generator reads the netlist and emits semantically equivalent C code — no guessing, no hand-coding.

### 2. Branchless Gate-Level Logic

All data-path operations are:
- **Gate cells only**: cell_and, cell_or, cell_mux, cell_addsub, etc.
- **No native C arithmetic** in the device tick: `+`, `-`, `*` forbidden
- **No conditionals**: No if/switch/?:. Selection is mask algebra.
- **Mask-algebra only**: Bitwise operations, shifts, zero-reduction equality.

### 3. Three-Phase Tick Discipline

Every `device_tick()` strictly observes:

```
READ PHASE         → Pre-read all consumed registers into const locals
COMPUTE PHASE      → All logic via cell calls (no native operators)
WRITE PHASE        → Commit all DFFs (no read-after-write)
```

This order enforces:
- No combinational loops
- Deterministic, cycle-accurate execution
- No hidden state changes mid-tick

### 4. Every Node is a Register

There is no "computed on the fly" signal. Every node — input, output, intermediate, state — is an addressed register slot. Wiring is placement; addresses define data flow.

## Netlist Specification

A valid `device.net.json` must contain:

```json
{
  "device": "name",
  "window_base": "0xADDR",
  "config_nodes": [ ... ],     // Static inputs (set once, never updated)
  "buffer_nodes": [ ... ],     // Dynamic inputs (presented externally each tick)
  "const_nodes": [ ... ],      // Constants (init once, never updated)
  "dff_nodes": [ ... ],        // Sequential state (updated each tick)
  "comb_nodes": [ ... ],       // Combinational logic (no state)
  
  "clock": { "counter": "...", "power": "...", "step": "..." },
  "pos": { "counter": "...", "increment": "..." },
  "writeout": { "enable": "...", "valid": "...", "lanes": [ ... ] },
  "display_ring": { "depth": N, "fields": [ ... ], "count": "..." }
}
```

## Supported Cell Types

### Combinational

- `buf` — buffer (passthrough)
- `not` — bitwise NOT
- `and` — bitwise AND
- `or` — bitwise OR
- `xor` — bitwise XOR
- `mux` — 2-to-1 multiplexer (mask algebra)
- `eqmask` — equality comparator (zero-reduction)
- `gate` — register-backed pass-or-zero (power gating)
- `fa` — full adder bit
- `addsub` — 64-bit ripple-carry add/subtract (gate netlist)
- `cmp_le` — unsigned less-than-or-equal (gate netlist)
- `cmp_lt` — unsigned less-than (gate netlist)
- `cmp_ge` — unsigned greater-than-or-equal (gate netlist)
- `cmp_gt` — unsigned greater-than (gate netlist)

### Sequential

- `dff` — D flip-flop with enable (implicit, via WRITE phase)

## Generated Functions

Every `device_gen.h` exports:

### `void device_init(word_t *r)`
Synchronous reset: zeroes all registers. Const nodes are initialized to their spec values.

### `void device_tick(word_t *r)`
One clock edge: READ→COMPUTE→WRITE. No external parameters; all state in `r[]`.

### `void device_run(word_t *r)`
Free-running loop: calls `device_tick()` repeatedly while power bit is set (if a power node exists). Otherwise, executes one tick and returns.

## Limitations and Notes

1. **Display Ring Field Mapping**: The display ring field names are assumed to match register names (lower-cased). If a field is derived (e.g., `ADP_CLK_AT_EMIT`), manually edit the WRITE phase or extend the netlist with explicit mappings.

2. **No Special Bus Nodes**: The generator processes only `config_nodes`, `buffer_nodes`, `const_nodes`, `dff_nodes`, and `comb_nodes`. Bus nodes (in passive devices like the wire) are not addressed registers — they live in a separate window.

3. **Determinism**: The generated code is byte-deterministic: running gennet_device.py on the same netlist always produces identical output.

```makefile

	gcc -o test_device test_device.c device.o
	./test_device

validate:
	python3 validate.py device.net.json
```
