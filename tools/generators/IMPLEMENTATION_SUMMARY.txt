## Objective

Build a tool that reads a `PARTS_LIST.md` file (human-readable hardware specification) and outputs `gen_*_net.py` (a Python emitter script that generates a declarative netlist in JSON).

## Architecture & Design

### Parser Strategy

The parser uses **regex-based extraction** to handle Markdown sections:

## <NUMBER>. <SECTION_TITLE>
  ├─ `NAME` — description
  ├─ `NAME[INDEX]` — description
  └─ `NAME[RANGE]` — description

**Section mapping:**
- §1 INPUT INTERFACE → `config_nodes`
- §2 PRICE-INDEXED TABLES → `tables`
- §3 BEST-PRICE → `dff_nodes`
- §4 RUNNING TOTALS → `dff_nodes`
- §6 EVENT COUNTERS → `counters`
- §7 METADATA → `dff_nodes`
- §8 RELAY OUTPUTS → `relay_nodes`
- §9 INTERNAL STATE → `dff_nodes`

**Type inference:**
- Contains 'strobe', 'flag', 'empty', 'valid', 'enable' → "type": "bit"
- Otherwise → "type": "u64"

### Generated Emitter Structure

#!/usr/bin/env python3
"""gen_<device>_net.py — EMITTER: writes <device>.net.json."""

import json

NET = {
    "device": "<device>",
    "window_base": "0x...",
    "kind": "...",
    "comment": "...",
    "config_nodes": [...],
    "dff_nodes": [...],
}

def main():
    with open("<device>.net.json", "w") as f:
        json.dump(NET, f, indent=2)
    print(f"emitted <device>.net.json")
