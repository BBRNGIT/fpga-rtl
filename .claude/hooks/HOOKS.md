# Git Hooks

This directory documents the pre-commit hooks that enforce fpga-rtl project policies.

## Active Hooks

### `.git/hooks/pre-commit`

**Purpose:** Enforce project commit and code policies before commits are created.

**Checks (in order):**

1. **AI Attribution** — Rejects commits with AI co-authoring markers
   - Detects: `Co-Authored-By: Claude`, `GENERATED`, "generated by AI"
   - Action: FAIL — commit blocked
   - Why: FOUNDER_VISION.md §11 — strict no-AI-attribution policy

2. **Vault Immutability** — Prevents modifications to `.hft/` directory
   - Detects: Any file in `.hft/` marked as modified or deleted
   - Action: FAIL — commit blocked
   - Why: FOUNDER_VISION.md §8 — vault is write-once
   - Solution: Create new versioned path (e.g., `adapter_v2/`)

3. **Thin Test Size** — Ensures test files are ≤45 lines
   - Detects: `test_*.c` files with >45 lines
   - Action: FAIL — commit blocked
   - Why: FOUNDER_VISION.md §9 — thin tests only (power-on + display)
   - Solution: Remove testbench logic; keep only init and display

4. **Generated File Compliance** — Warns if `*_gen.h` doesn't match gennet output
   - Detects: `*_gen.h` byte-differs from regenerating netlist
   - Action: WARN — commit proceeds, but message printed
   - Why: Build-sequence law — generated files should be reproducible
   - Solution: Regenerate from netlist before committing

## How Hooks Are Invoked

Hooks run automatically when you commit:

```bash
git commit -m "feat: add adapter"
# → Pre-commit hook runs
# → If any check fails: commit blocked, error printed
# → If all checks pass: commit proceeds
```

## Handling Hook Failures

### Failure: AI Attribution Detected

```
[FAIL] Commit message contains AI attribution.
       This project rejects AI co-authoring markers.
       
       FIX:
       1. Edit the commit message: git commit --amend
       2. Remove AI markers (Co-Authored-By, GENERATED, etc.)
       3. Try commit again
```

**Steps:**
```bash
git commit --amend                    # Edit message
# Remove any AI markers from the editor
# Save and exit
git commit -m "feat: corrected message"  # Try again
```

### Failure: Vault Immutability Violated

```
[FAIL] Cannot modify or delete files in .hft/ (immutable vault).

       Attempted modifications:
       (modified) .hft/adapter/adapter.net.json
       
       SOLUTION:
       Create a new versioned path instead.
         .hft/adapter/       (original, locked)
         .hft/adapter_v2/    (new version)
```

**Steps:**
```bash
git reset HEAD .hft/adapter/*        # Unstage vault changes
git checkout .hft/adapter/*          # Discard them

# Start over in staging
cp -r .hft/adapter .hft_staging/adapter_v2
cd .hft_staging/adapter_v2
# Make changes here, then graduate
```

### Failure: Test File Too Large

```
[FAIL] Test file exceeds 45 lines (thin test philosophy).

       File: .hft_staging/adapter/test_adapter.c
       Lines: 87 (max: 45)
       
       SOLUTION:
       1. Remove testbench logic (stepping, replay, etc.)
       2. Keep only: init, power-on loop, display
       3. Re-commit
```

**Steps:**
```bash
vim .hft_staging/adapter/test_adapter.c
# Remove testbench logic, keep lean test
# Save

wc -l .hft_staging/adapter/test_adapter.c  # Verify ≤45 lines
git add .hft_staging/adapter/test_adapter.c
git commit -m "fix: lean test"
```

### Warning: Generated File Mismatch

```
[WARN] Generated file may not match netlist (regenerate before commit).
       File: adapter_gen.h
       Netlist: adapter.net.json
       
       FIX:
       1. cd .hft_staging/adapter
       2. python3 gennet.py adapter.net.json > adapter_gen.h
       3. git add adapter_gen.h
       4. Try commit again
```

**Steps:**
```bash
cd .hft_staging/adapter
python3 gennet.py adapter.net.json > adapter_gen.h
git add adapter_gen.h
git commit -m "fix: regenerate adapter_gen.h"
```

## Bypassing Hooks (Not Recommended)

If absolutely necessary, you can bypass hooks:

```bash
# Skip pre-commit hook ONLY (not recommended)
git commit --no-verify -m "message"

# Note: This is discouraged and should only be used with founder approval
# for emergency fixes. Bypassed commits still need to pass gate.sh before
# graduation.
```

## Disabling Hooks

To temporarily disable hooks (for debugging):

```bash
# Disable pre-commit hook temporarily
chmod -x .git/hooks/pre-commit

# Re-enable
chmod +x .git/hooks/pre-commit
```

**Caution:** Only disable hooks for debugging. Re-enable before committing code changes.

## Testing Hooks Locally

To test hooks without committing:

```bash
# Run the pre-commit hook manually
bash .git/hooks/pre-commit

# Or test a specific check
bash -x .git/hooks/pre-commit 2>&1 | grep -A5 "AI Attribution"
```

## Hook Implementation Details

### Hook Location
```
.git/hooks/pre-commit       # Main enforcement hook
```

### Hook Logic

**Bash script** that:
1. Parses commit message (from `$1` argument or git log)
2. Checks staged files (via `git diff --cached`)
3. Validates policies
4. Reports violations with clear instructions
5. Exits with code 0 (pass) or 1 (fail)

### Adding New Checks

To add a new hook check:

1. Edit `.git/hooks/pre-commit`
2. Add a new section (following the existing pattern)
3. Use `git diff --cached` to inspect staged changes
4. Print clear error message with fix instructions
5. Exit 1 on failure
6. Test: `bash .git/hooks/pre-commit`

## Common Questions

**Q: Why block AI attribution?**
A: FOUNDER_VISION.md §11 — Project rejects AI co-authoring markers. Work is attributed to the person who directed and approved it.

**Q: Why block .hft/ edits?**
A: FOUNDER_VISION.md §8 — Vault is immutable (write-once). Prevents accidental overwrites of graduated components. New versions must use new paths.

**Q: Why enforce thin tests?**
A: FOUNDER_VISION.md §9 — Tests must be autonomous (power-on + display). No orchestration, stepping, or replay. Device runs on its clock alone.

**Q: What if I genuinely need to override a hook?**
A: Contact the project founder. Use `--no-verify` only with approval. Be aware that `gate.sh` will still validate before graduation.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Hook doesn't run | Check executable: `chmod +x .git/hooks/pre-commit` |
| Hook fails on clean commit | Check git config: `git config user.name` should not have "Claude" |
| False positive on .hft/ | Verify file is really in `.hft/`, not `.hft_staging/` |
| Test size hard to measure | Use `wc -l test_*.c` to count lines |
| Gennet doesn't regenerate deterministically | Check for randomness in gennet.py (sort dict keys, etc.) |
