# No AI Attribution

**Core Rule (§11 — Non-Negotiable):** This project rejects all AI co-authoring markers. Do not include:
- ❌ `Co-Authored-By: Claude ...`
- ❌ `GENERATED` comments
- ❌ Comments about AI or Claude
- ❌ Any AI attribution in commits, code, or documentation

**Commits carry the user's name only. This is enforced and non-negotiable.**

## Why This Matters

1. **Authorship clarity:** Work is attributed to the person who directed and approved it
2. **Legal responsibility:** The person committing is responsible for the code
3. **Project integrity:** Clear authorship prevents confusion in audits and version control
4. **No AI branding:** The system is hardware-focused, not AI-focused

## What NOT to Do

### ❌ Commits with Co-Authored-By

```bash
# WRONG
git commit -m "fix: critical bug

Co-Authored-By: Claude Haiku <noreply@anthropic.com>"
# Blocked by commit hooks
```

### ❌ GENERATED Comments in Code

```c
// WRONG — in adapter.c
// GENERATED BY CLAUDE — DO NOT EDIT

void adapter_tick(...) { ... }
```

### ❌ AI Attribution in Commit Messages

```bash
# WRONG
git commit -m "feat: add adapter — written with Claude Code"
# Should just be:
# "feat: add adapter — price feed deserialization"
```

### ❌ Docstrings Mentioning AI

```c
// WRONG — in adapter.h
/**
 * Adapter module generated by Claude.
 * Uses AI-guided netlist generation.
 */
```

## What TO Do

### ✅ User-Only Attribution

```bash
# CORRECT
git commit -m "feat: add adapter — price feed deserialization"
# Author: Test User <user@example.com>
# No AI markers in commit message or code
```

### ✅ Code Comments (Domain-Focused)

```c
// CORRECT — in adapter.c
// Read price from wire, validate, output clean stream
// READ: wire bid/ask
// COMPUTE: validate prices (cell_cmp_gt)
// WRITE: latch to output

static inline void adapter_tick(word_t *r) {
  ...
}
```

### ✅ Docstrings (Architecture-Focused)

```c
// CORRECT — in adapter.h
/**
 * Adapter — price feed adapter
 * 
 * Reads raw market prices from wire bus.
 * Validates bid > 0 and ask > 0.
 * Outputs clean (bid, ask, seq) to pipeline.
 * 
 * Architecture: READ→COMPUTE→WRITE
 * Generates on every MAC edge.
 */
```

## Enforcement

### Pre-commit Hook

The project has (or will have) a pre-commit hook that:

1. **Scans commit message** — Rejects `Co-Authored-By`, `GENERATED`, "Claude", "AI"
2. **Scans files** — Rejects GENERATED comments, AI attribution in docstrings
3. **Blocks commit** — If violations found

**Expected output on violation:**
```
[FAIL] Commit message contains AI attribution: "Co-Authored-By: Claude"
       This project rejects AI co-authoring markers.
       Use user name only: git config user.name "Your Name"
```

### Git User Configuration

Ensure your git user is set correctly:

```bash
git config user.name "Your Name"
git config user.email "your@email.com"

# Verify
git config --list | grep user
# user.name=Your Name
# user.email=your@email.com
```

## Commit Message Guidelines

**Format:** `<type>: <description>`

```bash
# CORRECT examples:
git commit -m "feat: add adapter — price feed deserialization"
git commit -m "fix: gate-level arithmetic check — handle shifts correctly"
git commit -m "docs: improve CLAUDE.md — clarify gate.sh stages"
git commit -m "refactor: netlist validator — split single-writer check"
```

**Guidelines:**
- Type: feat, fix, docs, refactor, test, chore
- Description: lowercase, past tense, specific (not "improve", say "improve what")
- No AI mention, no "written by", no "generated"
- No `Co-Authored-By`

## Documentation and Code Comments

### ✅ In CLAUDE.md, design docs, component READMEs

```markdown
# Adapter Component

The adapter reads price data from the wire and outputs clean (bid, ask, seq) records.

- Reads: WIRE_BID, WIRE_ASK
- Outputs: ADAPTER_PRICE, ADAPTER_READY
- Architecture: Price validation via cell-level gates
- Generated: netlist via gen_adapter_net.py, device C via gennet.py

[No mention of AI or Claude]
```

### ✅ In code (READ→COMPUTE→WRITE patterns)

```c
static inline void adapter_tick(word_t *r) {
  // READ phase
  uint64_t bid = r[WIRE_BID];
  uint64_t ask = r[WIRE_ASK];
  
  // COMPUTE phase: validate via cell-level gates
  uint64_t bid_valid = 0;
  cell_cmp_gt(bid, 0, &bid_valid);
  
  // WRITE phase
  r[ADAPTER_VALID] = bid_valid;
  r[ADAPTER_PRICE] = bid;
}
```

## When Using Claude Code (This Tool)

**Internally, the tool may show attribution.** That's fine for development. But:

1. **Before committing:** Review commits for AI markers
2. **Remove any traces:** Change Co-Authored-By, remove GENERATED comments
3. **Commit with user name only:** `git commit` (not with AI attribution)

**Example cleanup:**
```bash
# The tool generated a commit with Co-Authored-By
git log -1
# [shows: Co-Authored-By: Claude Haiku]

# Fix it before pushing
git reset --soft HEAD~1  # Undo commit
git add .
git commit -m "same message, no AI attribution"

# Now safe to push
```

## Checking for Violations

Before pushing:

```bash
# Check recent commits for AI markers
git log -10 --format="%B" | grep -i "claude\|ai attribution\|generated"
# Should return nothing

# Check code for GENERATED comments
grep -r "GENERATED\|claude\|anthropic" .hft_staging --include="*.c" --include="*.h"
# Should return nothing (except this file)

# Check for Co-Authored-By
git log -10 --format="%B" | grep "Co-Authored-By"
# Should return nothing
```

## If You Find Violations

**In a commit already pushed:**

1. **Notify the team** — Explain the issue
2. **Create a new commit** — Fix the code/message
3. **Re-commit** — With corrected attribution

```bash
# Example: Remove GENERATED comment
vim adapter.c  # Remove "// GENERATED BY CLAUDE" line
git add adapter.c
git commit -m "fix: remove ai attribution comment"
git push
```
