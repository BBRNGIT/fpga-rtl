# Thin Test Philosophy

**Core Rule:** Device tests are **thin**: power-on and display only. No external orchestration, no stepping, no data injection, no CSV/replay tokens in the device address map.

**Why:** The device runs autonomously on its clock. External control would require adding testbench logic to the device (violating purity). Instead, test on real data externally prepped.

## What a Thin Test Is

```c
// test_adapter.c — thin test
#include "adapter.h"
#include "data_synth.bin"  // Prepped data

int main() {
  word_t *r = malloc_backplane();
  
  // Power on (boot the device)
  for (int cycle = 0; cycle < 1000; cycle++) {
    adapter_tick(r);      // Device runs on its clock
  }
  
  // Display results
  printf("ADAPTER_PRICE: %lx\n", r[ADAPTER_PRICE]);
  printf("ADAPTER_READY: %lx\n", r[ADAPTER_READY]);
  
  return 0;
}
```

**Size:** ≤45 lines (enforced by gate.sh).

## What a Thin Test Is NOT

❌ **Not an orchestrator:**
```c
// WRONG — injecting data mid-run
for (int i = 0; i < 100; i++) {
  r[INPUT] = test_data[i];  // Injection
  adapter_tick(r);
  if (r[OUTPUT] != expected[i]) {
    printf("FAIL at step %d\n", i);  // Stepping
  }
}
```

❌ **Not a replay harness:**
```c
// WRONG — replay concepts in device
while (replay_read(&pkt)) {
  r[INPUT_PACKET] = pkt;
  adapter_tick(r);
}
```

❌ **Not a testbench with tokens:**
```c
// WRONG — CSV, file, source-type in device registers
r[CSV_FILE] = "data.csv";
r[SOURCE_TYPE] = REPLAY_SOURCE;
adapter_tick(r);
```

## The Test Structure

### 1. Setup (Prepped Data)

Data is prepared **offline** before the test runs:

```sh
# prep.py — convert CSV to binary
python3 prep.py data.csv > data.bin
# Or: data_synth.bin (generated synthetic data)
```

### 2. Initialize

Boot the device (power-on sequence):

```c
int main() {
  word_t *r = malloc_backplane();
  memset(r, 0, backplane_size);  // Zero initialization
  
  // Power up: run a few ticks to settle (optional)
  for (int i = 0; i < 10; i++) {
    adapter_tick(r);
  }
```

### 3. Run

Device runs autonomously on its clock (no external stepping):

```c
  // Device processes prepped data autonomously
  for (int cycle = 0; cycle < num_cycles; cycle++) {
    adapter_tick(r);
    // No inspection, no stepping
  }
```

### 4. Observe

Read and display final state:

```c
  // Display final results
  printf("FINAL STATE:\n");
  printf("  ADAPTER_PRICE: %lx\n", r[ADAPTER_PRICE]);
  printf("  ADAPTER_READY: %lx\n", r[ADAPTER_READY]);
  printf("  ADAPTER_COUNT: %lx\n", r[ADAPTER_COUNT]);
  
  return 0;
}
```

## Thin Test Enforcement

`gate.sh` stage 2 builds and runs the thin test:

```sh
# In Makefile
test: $(DEVICE_TEST)
	./$(DEVICE_TEST)

$(DEVICE_TEST): test_adapter.o adapter_gen.h
	gcc -o $(DEVICE_TEST) test_adapter.c
	./$(DEVICE_TEST)
```

**Output:**
```
FINAL STATE:
  ADAPTER_PRICE: 3a9800
  ADAPTER_READY: 1
  ADAPTER_COUNT: 3e8
```

**Requirements:**
- Test must exit cleanly (exit 0)
- Completes in < 1 second (reasonable performance)
- ≤45 lines (enforced by pre-commit hook, future)

## Example Thin Test

```c
// test_adapter.c (real example)
#include "adapter.h"
#include <stdio.h>
#include <string.h>

// Prepped binary data (offline CSV conversion)
extern uint8_t data_synth_bin[];
extern size_t data_synth_bin_len;

int main() {
  // Allocate backplane
  word_t *r = malloc(sizeof(word_t) * BACKPLANE_SIZE);
  memset(r, 0, sizeof(word_t) * BACKPLANE_SIZE);
  
  // Load prepped data into INPUT register
  // (data_synth.bin was generated by prep.py offline)
  size_t offset = 0;
  while (offset < data_synth_bin_len) {
    // Each cycle, copy next input packet
    memcpy(&r[INPUT_PACKET], &data_synth_bin[offset], sizeof(packet_t));
    offset += sizeof(packet_t);
    
    // Run device for this cycle
    adapter_tick(r);
  }
  
  // Display final state
  printf("ADAPTER_PRICE: %016lx\n", r[ADAPTER_PRICE]);
  printf("ADAPTER_READY: %d\n", r[ADAPTER_READY] & 1);
  printf("ADAPTER_SEQ: %d\n", (int)r[ADAPTER_SEQ]);
  
  free(r);
  return 0;
}
```

## Prepped Data (offline)

Data is prepared **before** test runs. No CSV or file tokens in the device:

```python
# prep.py — offline data preparation
import struct
import csv

def prep_csv(input_csv, output_bin):
    with open(input_csv) as f:
        reader = csv.DictReader(f)
        with open(output_bin, 'wb') as out:
            for row in reader:
                bid = int(row['bid'])
                ask = int(row['ask'])
                packet = struct.pack('<QQ', bid, ask)
                out.write(packet)

prep_csv('data.csv', 'data.bin')
```

**Result:** data.bin is binary, committed to repo. Test loads it at startup.

## No Testbench Tokens in Device

The device address map must **not** contain:

❌ CSV_FILE
❌ REPLAY_SOURCE
❌ SOURCE_TYPE
❌ RECORD_MODE
❌ STEP (stepping command)
❌ HALT (pause command)
❌ Any orchestration register

The device is **autonomous**; it runs on its clock alone.

## Probe (Extended Observation)

For detailed debugging, use `probe.c` (not a thin test, but allowed):

```c
// probe.c — per-cycle waveform dump
#include "adapter.h"

int main() {
  word_t *r = malloc(backplane_size);
  memset(r, 0, backplane_size);
  
  // Open waveform output
  FILE *wave = fopen("adapter.vcd", "w");  // VCD format
  
  for (int cycle = 0; cycle < num_cycles; cycle++) {
    // Write waveform before tick
    fprintf(wave, "#%d\n", cycle);
    fprintf(wave, "$BID %016lx\n", r[WIRE_BID]);
    fprintf(wave, "$ASK %016lx\n", r[WIRE_ASK]);
    
    // Run device
    adapter_tick(r);
    
    // Write waveform after tick
    fprintf(wave, "$OUT %016lx\n", r[ADAPTER_PRICE]);
  }
  
  fclose(wave);
  return 0;
}
```

**Usage:**
```sh
make probe  # Generate waveform
# Then inspect adapter.vcd with waveform viewer
```

## Pre-graduation Checklist

Before `gate.sh`:

- [ ] Test file ≤45 lines
- [ ] Test compiles without errors
- [ ] Test runs in < 1 second
- [ ] Test exits cleanly (exit 0)
- [ ] No stepping, replay, CSV tokens in device registers
- [ ] Data is prepped offline (not injected mid-test)
- [ ] No orchestration in test; device runs autonomously

## Common Test Violations

| Violation | Fix |
|-----------|-----|
| Injecting data mid-tick | Prep data offline; load at startup |
| Stepping through cycles | Device runs continuously; don't pause |
| Replay harness in device | Orchestration goes outside device; device is autonomous |
| CSV file token in registers | Pre-convert CSV to binary offline |
| Testbench logic in device | Keep device pure; test harness is separate |
