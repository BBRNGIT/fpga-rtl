# Build-Sequence Law Enforcement

**Core Rule:** Device logic is GENERATED ONLY. Hand-written `.c` and `.h` files must NOT contain:
- `cell_*()` function calls
- `*_tick()` function definitions

All device logic lives in `*_gen.h` (generated by gennet from the netlist).

## What This Means

### ❌ NOT Allowed

```c
// BAD — in adapter.c or adapter.h
#include "adapter.h"

static inline void adapter_tick(word_t *r) {  // ❌ Hand-written tick
  r[PRICE_OUT] = r[PRICE_IN] + 100;           // ❌ Native arithmetic
  cell_mux(r[SEL], r[A], r[B], &r[MUX_OUT]); // ❌ Hand-written cell call
}
```

### ✅ ALLOWED

```c
// GOOD — in adapter_gen.h (generated by gennet.py)
static inline void adapter_tick(word_t *r) {
  // READ phase
  uint64_t in_val = r[PRICE_IN];
  uint64_t sel = r[SEL];
  
  // COMPUTE phase (generated from netlist)
  uint64_t sum_out = 0;
  cell_addsub(in_val, 100, 0, &sum_out);
  
  uint64_t mux_out = 0;
  cell_mux(sel, r[A], r[B], &mux_out);
  
  // WRITE phase
  r[PRICE_OUT] = sum_out;
  r[MUX_OUT] = mux_out;
}
```

## The Build-Sequence Law

**Enforcement chain:**

1. **You write:** `gen_<module>_net.py` (emitter that produces netlist)
2. **You write:** `gennet.py` (generator that produces device C)
3. **System generates:** `<module>.net.json` (from emitter)
4. **System generates:** `<module>_gen.h` (from gennet)
5. **You commit both** generated files as proof of specification
6. **gate.sh stage 2c checks:** No hand-written device logic exists

## gate.sh Stage 2c: Build-Sequence Check

The gate scans all `.c` and `.h` files (excluding `cells.h` and `*_gen.h`) for violations:

```sh
# In gate.sh (simplified)
for SRC in *.c *.h; do
  case "$SRC" in cells.h|*_gen.h) continue;; esac
  
  # Look for cell calls or tick definitions
  grep -E 'cell_(buf|not|and|or|xor|mux|eqmask|fa|gate|sar|dff|addsub)' "$SRC"
  grep -E '^[[:space:]]*void [a-z_]+_tick' "$SRC"
done
```

**Output on violation:**
```
[FAIL] adapter.c — hand-written device logic; write the netlist + emitter, run gennet:
  123: cell_addsub(a, b, 0, &result);
  124: cell_mux(sel, x, y, &mux_out);
```

## How to Verify Compliance

### Check 1: No cell_*() in hand-written files

```sh
cd .hft_staging/<module>
grep -r "cell_" *.c *.h 2>/dev/null | grep -v "^[^:]*:.*//.*cell_"
# Should return nothing
```

### Check 2: No *_tick definitions in hand-written files

```sh
cd .hft_staging/<module>
grep -E "void [a-z_]+_tick\(" *.c *.h | grep -v "_gen.h"
# Should return nothing
```

### Check 3: Verify *_gen.h is generated

```sh
# Regenerate and compare
python3 gennet.py <module>.net.json > <module>_gen.h.new
diff <module>_gen.h <module>_gen.h.new
# Should be identical (or show only timestamp differences)
```

## Common Violations and Fixes

| Violation | File | Fix |
|-----------|------|-----|
| `cell_addsub()` call | adapter.c | Move logic to netlist (comb_nodes); regenerate |
| `void adapter_tick()` def | adapter.h | Delete tick; let gennet generate it |
| Inline mux logic | buffer.c | Only conditionals for external ingress allowed here |
| `cell_dff()` call | display.c | Only in `*_gen.h`; don't write state machines manually |

## What IS Allowed in Hand-Written Code

✅ **buffer.c** — External data ingestion and buffering (conditionals allowed here)
✅ **display.c** — Output formatting and display logic (read-only access)
✅ **probe.c** — Diagnostic waveform dumping
✅ **test_*.c** — Thin test (power-on + display only)
✅ ***.c/.h** — Host glue, initialization, non-device code

## Correct Development Flow

```
1. Design in <MODULE>.md
2. Write gen_<module>_net.py (emitter)
3. Run: python3 gen_<module>_net.py > <module>.net.json
4. Validate: python3 validate.py <module>.net.json
5. Write gennet.py (generator)
6. Run: python3 gennet.py <module>.net.json > <module>_gen.h
7. #include "<module>_gen.h" in main driver
8. Write buffer.c, display.c (non-device logic)
9. Run: make test
10. Run: gate.sh (checks for violations)
11. If gate passes, commit and graduate
```

## Why This Rule Matters

1. **Reproducibility:** Running gennet on committed netlist must byte-match committed `*_gen.h`
2. **Architecture enforcement:** Device logic is data-flow, not imperative
3. **Testability:** Gate.sh stage 2c proves the circuit came from specification, not ad-hoc coding
4. **Clean-room builds:** No hand-written logic means no hidden artifacts

## Pre-graduation Checklist

Before running `gate.sh`:

- [ ] `*_gen.h` is in the repo (not gitignored)
- [ ] `*.net.json` is committed
- [ ] `gennet.py` exists and is hand-written
- [ ] No `cell_*()` calls in `.c`/`.h` (except `cells.h` and `*_gen.h`)
- [ ] No `*_tick()` definitions in `.c`/`.h` (except `*_gen.h`)
- [ ] All hand-written tick logic deleted or moved to netlist
